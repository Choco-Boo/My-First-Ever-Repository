Git Commands

Mainly about saving snapchots and checking logs and status of files

    • git init - Used to initialize a Git repository
      git add FileName or git add . - this is to add either file or everything to staging
      git commit -m "Message" - Message/note regarding this commit to the local repo
      git status - to check the status of the working directory and staging area
      git log - to view the commit history of the repository

    • Reverting to old versions

    • git checkout "id/name of commit" - Switches to a specific commit in the repository, so it's like going back in time

    Note regarding git checkout

    git checkout is used to switch branches or restore working tree files, but it can also be used to view the state of the repository at a specific commit.
    When you checkout a commit, you are in a "detached HEAD" state, which means you are not on any branch. This allows you to explore the repository's history without affecting the current branch.

    To return to the latest commit on the current branch, you can use "git checkout branch-name" or "git checkout master/main" (if your main branch is named "master or main").
