Git Commands

Mainly about saving snapshots and checking logs and status of files

  1. git init - Used to initialize a Git repository
  2. git add FileName or git add . - Add a file or everything to staging
  3. git commit -m "Message" - Record changes to the repository with a message
  4. git status - Check the status of the working directory and staging area
  5. git log - View the commit history of the repository

Working with the Branch

  • git checkout "id/name of commit" - Switches to a specific commit in the repository (detached HEAD),
    allows you to move to a certain commit in your current branch
  • git checkout branch-name - Switches to the specified branch
  • git switch branch-name - Modern way to switch to a specified branch
  • git branch - Lists all branches in the repository

Advance branch commands

  • git branch -d branch-name - Deletes the specified branch (only if it has been merged)
  • git branch -D branch-name - Force deletes the specified branch (even if it hasn't been merged)

Note regarding git checkout

git checkout is used to switch branches or restore working tree files, but it can also be used to view the state of the repository at a specific commit. When you checkout a commit, you are in a "detached HEAD" state, which means you are not on any branch. This allows you to explore the repository's history without affecting the current branch.

To return to the latest commit on the current branch, you can use "git checkout branch-name" or "git checkout master/main" (if your main branch is named "master" or "main").

How to create branches

There are many ways of making branches; you can make them out of other branches or out of commits.

  1. git branch "name of branch" - Creates a new branch with the given name, but keeps you on your current branch
  2. git checkout -b "name of branch" - Creates a new branch and switches to it immediately
  3. git switch -c "name of branch" - Modern way to create a new branch and switch to it

GitHub Basics

GitHub is a web-based platform that provides hosting for Git repositories. It offers a range of features for collaboration, version control, and project management. Here are some basic GitHub commands and concepts:

  • git clone - Used to create a local copy of a remote repository on GitHub. It allows you to work on the project locally and sync changes with the remote repository.
  • git push - Used to upload your local commits to a remote repository on GitHub. It allows you to share your changes with others and update the remote repository.
  • git pull - Used to fetch and merge changes from a remote repository on GitHub into your local repository. It allows you to stay up-to-date with the latest changes made by others.
  • git fork - A feature on GitHub that allows you to create a personal copy of someone else's repository. It enables you to make changes and propose them back to the original repository through pull requests.

GitHub also provides a web interface for managing repositories, issues, pull requests, and more. It is widely used for open-source projects, collaboration, and version control in software development.

GitHub "Pushing" - Local Repository

To push your local commits to a remote repository on GitHub, you can use the following command:

  1. git remote add origin URL - Adds a remote repository with the name "origin" and the specified URL
  2. git branch -M main - Renames the current branch to "main" (if your main branch is named "master", you can skip this step)
  3. git push -u origin main - Pushes your local commits to the "main" branch of the remote repository and sets up tracking
  4. git push - After the initial push, you can simply use "git push" to push subsequent commits to the remote repository
  5. git pull origin main - Fetches and merges changes from the "main" branch of the remote repository into your local repository
  6. git push origin branch-name - Pushes the specified branch to the remote repository
